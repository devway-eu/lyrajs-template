## Contributor License Agreement (CLA)

Before we can accept your contribution, you must sign our Contributor License Agreement (CLA). This ensures that:

- You grant us the rights to use your contribution
- We can maintain flexibility to relicense the project in the future
- The project can offer commercial licenses alongside the open source version

**How to sign:**
1. When you submit your first pull request, our CLA bot will comment with instructions
2. Simply follow the link and sign electronically (takes 1 minute)
3. Once signed, all your future contributions are covered

## How to Contribute

### 1. Fork & Clone
```bash
git clone https://github.com/yourusername/lyrajs-core.git
cd lyrajs-core
npm install
```

### 2. Create a Branch
```bash
git checkout -b feature/your-feature-name
```

### 3. Make Your Changes
- Write clean, readable code
- Follow the existing code style
- Add tests if applicable
- Update documentation if needed

### 4. Test Your Changes
```bash
npm run build
npm test  # if tests exist
```

### 5. Commit
```bash
git add .
git commit -m "feat: description of your changes"
```

Use conventional commit format:
- `feat:` for new features
- `fix:` for bug fixes
- `docs:` for documentation
- `refactor:` for code refactoring
- `test:` for adding tests

### 6. Push & Create PR
```bash
git push origin feature/your-feature-name
```

Then create a Pull Request on GitHub.

## Code Style

- Use TypeScript
- Follow existing patterns in the codebase
- Use meaningful variable names
- Add comments for complex logic

## Questions?

Open an issue or reach out to the maintainers.
